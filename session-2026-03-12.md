# Session TDBR site_v3 — 2026-03-12

## Modifications apportées

### 1. Import maquettes Printful pour articles existants
**Fichier :** `src/Controller/Admin/PrintfulAdminController.php`  
**Commit :** `16b9482`

**Problème :** Quand un article existait déjà en base, les maquettes Printful n'étaient jamais importées même si la case "Import mockups" était cochée.

**Solution :** Dans le bloc de traitement des articles existants, ajout de la logique d'import différentiel :
- Récupère les URLs des images déjà présentes sur l'article
- Pour chaque maquette Printful, n'importe que celles dont l'URL est absente
- Les nouvelles images sont ajoutées à la suite avec un ordre incrémental
- Aucun doublon possible (comparaison par URL)

---

### 2. Drag & drop pour réordonner les maquettes (page admin article)
**Fichier :** `templates/admin/articles/form.html.twig`  
**Commit :** `432e3a4`

**Fonctionnement :** Drag & drop natif HTML5 via Alpine.js, sans dépendance externe.

**Détails :**
- Ajout de `dragIndex`, `dragStart()`, `dragOver()`, `dragEnd()` dans l'Alpine data
- Chaque carte image est `draggable="true"` avec les handlers appropriés
- L'image glissée devient semi-transparente (opacity 35%) pendant le déplacement
- Icône `⠿` en bas à droite pour indiquer les éléments déplaçables
- Message d'aide affiché quand il y a plus d'une image
- La clé `x-for` des hidden inputs passe de l'index à l'URL (stable pendant le réordonnancement)
- L'ordre du tableau `images[]` est soumis automatiquement avec le formulaire → `ordre` en base

---

### 3. Réactivation du paiement Stripe sur le checkout
**Fichier :** `templates/public/checkout.html.twig`  
**Commit :** `bf3dd48`

**Contexte :** Le paiement avait été volontairement désactivé côté UI (message "temporairement indisponible", bouton disabled). Le backend (StripeService, webhook, PaymentIntent) était resté intact et fonctionnel.

**Modifications :**
- Remplacement du message d'indisponibilité par `<div id="payment-element">` (point de montage Stripe Elements)
- Ajout de `<div id="payment-message">` pour les erreurs Stripe
- Rétablissement du bouton de paiement : `@click="pay()"`, montant dynamique via Alpine (`formatPrice(grandTotal)`), spinner pendant le traitement

**Prérequis serveur :** Les vraies clés Stripe doivent être renseignées dans `.env.local` :
```
STRIPE_PUBLIC_KEY=pk_live_...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
```
